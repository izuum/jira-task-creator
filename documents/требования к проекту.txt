Проект: Jira Task Creator

Цель: разработать программу на Java для автоматического создания задач в системе управления проектами и отслеживания заданий «Jira» на основе данных из YAML-файла.

Функциональные требования: 

ФТ-1: Чтение конфигурации подключения 
	- Для подключения к Jira, программа должна считать отдельный конфигурационный файл, в котором будет URL, логин, токен и ключ проекта. Файл не должен отслеживаться системой контроля версий.
	- Тип аутентификации Basic Auth(логин + токен)


ФТ-2: Валидация файла с задачами (Пример валидного файла: ПРИЛОЖЕНИЕ 1)
	-	Программа проверяет, существует ли файл listOfTask.yml по указанному пути
	-	Программа проверяет, чтобы файл был не пустой
	-	Программа парсит файл listOfTask.yml c помощью библиотеки Jackson
	-	Если файл отсутствует, необходимо вывести сообщение в лог-файл: [ERROR] File not found: <path> и завершить работу программы
	-	Если файл пуст или содержит некорректную структуру yml, необходимо вывести сообщение в лог-файл: [ERROR] Invalid YAML structure:
			- некорретной структурой считается синтаксическая ошибка в файле
			- если ошибка возникла при выполнении одной из задач описанных в файле, эту задачу следует пропустить и продолжить выполнение программы
	- 	Если отсутствует одно из обязательных полей в блоке с задачей, необхоидимо вывести сообщене в лог-файл: [ERROR] Required field is missing (<summary>)

ФТ-3: Маппинг данных
    - Необходимо десериализовать данные из YAML-файла в Java POJO класс

ФТ-4: Взаимодействие с Jira API:
    - Программа для каждой задачи из YAML-файла должны выполнить POST запрос к Jira REST API
    - Если задача создана, необходимо вывести сообщение в лог-файл в кодом ответа
    - Если POST запрос по какой-либо задаче не получилось выполнить, программа должна его пропустить и вывести сообщение в лог-файл с кодом ответа

Нефункциональные требования:
НТ-1: Ведение логов
    - Необходимо вести лог-файл сессии (например issues_DDMMYYYY.txt) со всеми ответами Jira с помощью Log4j, а именно:
			- если произошла ошибка чтения файла, необходимо в лог-файл записать сообщение с ошибкой [ERROR] Invalid YAML structure, либо [ERROR] File not found: <path>, в зависимости от того, какую ошибку выбросит программа
			- если выполнение запроса завершилось успешно, необходимо записать в лог-файл сообщение [SUCCESS] Task <summary> was created
			- если выполнение запроса завершилось с ошибкой, необходимо записать в лог-файл сообщение [FAILED] Task <summary> was not created
			- в случае получения ошибки от Jira (HTTP статус 4xx или 5xx), тело ответа также должно быть записано в лог-файл для диагностики причин отказа
	- Папка с лог-файлами должна лежать в корне проекта
НТ-2: Задержка между запросами
    - Необходима задержка между запросами, чтобы не получить rate limit от Jira
НТ-3: Технологический стек
    - Java 17, Rest Assured, Jackson, Lombok, Log4j. Система сборки Maven.
НТ-4: Структура yml-файлов
	credentials:
		jira_url: https://example-domain.atlassian.net (тип String)
		username: email@example.com (тип String)
		token: “api-token” (тип String)
		projectKey (необходим указать глобально в конфиге) (тип String)
	
	fields:
		summary: “Создать задачу в Jira” (тип String)
		issuetype: 
			name: “Task” (тип String)
		priority: 
			name: “High” (тип String)
		description: “Описание задачи” (тип String)

Требования к входным данным( в .yml файле):
	Обязательные поля:
		- summary (краткое описание)
		- type:
			name:(тип задачи: Bug, Task)
		- priority:
			name: (приоритет)
	Опциональные поля:
		- description (развернутое описание)
НТ-5: Endpoints необходимые для работы приложения
	1)	/rest/api/2/issue 
		Метод: POST
		Назначение: Создание новой задачи в Jira
	
Требования к завершению приложения:
	- Если в процессе работы программы возникнет ошибка об отсутствии файла с задачами, программа должна завершить работу
	- Если в процессе работы программы возникнет синтаксическая ошибка в одном из блоков задач, программа должна пропустить эту задачу и занести данные об ошибке в этом блоке в лог-файл
		 

ПРИЛОЖЕНИЕ:
1)	Пример валидного yml-файла с задачей:
fields:
		summary : “Создание Java POJO-класса для десериализации yml-файла с задачами”   
		issuetype : 
			name: “Task”
		priority : 
			name: “High”
		description :  “ Необходимо создать Java POJO-класс с полями соответствующими полям yml-файла с задачами, для того чтобы десериализовать данные о задаче и отправить их в Jira”

2)	Формат ошибок и info-сообщений
	- [ERROR] File not found: <path> - ошибка возникающая, если по прописанному пути нет	  .yml-файла с задачами, где <path> - ожидаемый путь к файлу
	- [ERROR] Invalid YAML structure - ошибка возникающая, если yml-файле в названии полей допущены синтаксическая ошибка
	- [ERROR] Required field is missing - ошибка возникающая, если отсутствует какое-либо обязательное поле
	- [SUCCESS] Task <summary> was created - информативное сообщение о том, что задача была создана, где <summary> - это краткое описание задачи
	- [FAILED] Task <summary> was not created - информативное сообщение о том, что задача не была создана по какой-либо причине, где <summary> - это краткое описание задачи
