Тест-кейсы к программе Jira Task Creator

Тестирование конфигурации:

	TC-Config-01 "Успешное чтение конфигурации":
		Шаги:
			1. создать валидный конфигурационный файл с корректными URL, логином, токеном и projectKey
			2. поместить его в корень проекта
			3. запустить программу с валидным listOfTask.yml
		
		Ожидаемый результат:
			1. программа успешно подключается к Jira
			2. в Jira создается новая задача
			3. в лог-файле появляется сообщение [SUCCESS] Task "Создание Java POJO-класса..." was created и код ответа 201(Created)
			
	TC-Config-02 "Проверка игнорирования конфигурации Git-ом":
		Шаги:
			1. проверить, добавлен ли файл конфигурации .gitignore
		
		Ожидаемый результат:
			1. файл с конфигурацией должен быть в списке .gitignore
			

Тестирование валидации YAML-файла с задачами:

	TC-Validation-01 "Файл с задачами не существует":
		Шаги:
			1. указать программе путь к несуществующему файлу listOfTask.yml
			2. запустить программу
			
		Ожидаемый результат:
			1. программа завершает работу
			2. в лог-файл выводится сообщение [ERROR] File not found: <path>
			
	TC-Validation-02 "Файл с задачей пустой":
		Шаги:
			1. создать пустой файл listOfTask.yml
			2. указать программе путь к пустому файлу listOfTask.yml
			3. запустить программу
		
		Ожидаемый результат:
			1. программа завершает работу
			2. в лог-файл выводится сообщение [ERROR] Invalid YAML structure
	
	TC-Validation-03 "Синтаксическая ошибка в YAML":
		Шаги:
			1. в файле listOfTask.yml намеренно сделать ошибку (например написать sumary, вместо summary)
			2. запустить программу
			
		Ожидаемый результат:
			1. программа пропускает эту задачу, но продолжает работу с остальными
			2. в лог-файл выводится сообщение [ERROR] Invalid YAML structure
			
	TC-Validation-04 "Отсутствует обязательное поле (summary)":
		Шаги:
			1. в файле listOfTask.yml удалить поле summary у одной из задач
			2. запустить программу
			
		Ожидаемый результат:
			1. программа пропускает эту задачу, но продолжает работу с остальными
			2. в лог-файл выводится сообщение [ERROR] Required field is missing (summary)
			
Тестирование создания задач через API:

	TC-API-01 "Успешное создание задачи":
		Шаги:
			1. подготовить файл listOfTask.yml с одной валидной задачей
			2. запустить программу
		
		Ожидаемый результат:
			1. программа успешно подключается к Jira
			2. в Jira создается новая задача
			3. в лог-файле выводится сообщение [SUCCESS] Task "Создание Java POJO-класса..." was created и код ответа 201(Created)
			
	TC-API-02 "Создание нескольких задач":
		Шаги:
			1. подготовить файл listOfTask.yml с тремя валидными задачами
			2. запустить программу
			
		Ожидаемый результат:
			1. все три задачи успешно создаются в Jira
			2. в лог-файле три записи [SUCCESS]
			3. проверить, что между отправкой задач была пауза 1-2 сек., чтобы не было rate limit
	
	TC-API-03 "Ошибка от Jira":
		Шаги:
			1. в конфигурационном файле указать неверный projectKey
			2. запустить проверить
			
		Ожидаемый результат:
			1. задача не создается
			2. в лог-файле выводится сообщение [FAILED] Task <summary> was not created и тело ответа от Jira
	
	TC-API-04 "Неверный типа задачи":
		Шаги:
			1. в задаче в поле issuetype.name указать "SuperTask"
			2. запустить программу
			
		Ожидаемый результат:
			1. задача не создается
			2. в лог-файле выводится сообщение [FAILED] Task <summary> was not created и тело ответа от Jira
			
Тестирование логгирования:
	
	TC-Log-01 "Проверка формата имени лог-файла":
		Шаги:
			1. запустить программу 25.02.2026
		
		Ожидаемый результат:
			1. в папке с логами должен создаться файл issue_25022026.txt
	
	TC-Log-02 "Проверка логирования ошибок валидации":
		Шаги:
			1. выполнить тесты TC-Validation-01, TC-Validation-02, TC-Validation-03, TC-Validation-04
		
		Ожидаемый результат:
			1. сообщения в лог-файле должны точно соответствовать форматам из требований: [ERROR]....
			
	TC-Log-03 "Проверка логирования успеха/неудачи":
		Шаги:
			1. выполнить тесты TC-API-01, TC-API-03
		
		Ожидаемый результат:
			1. сообщения в лог-файле должны точно соответствовать форматам из требований: [SUCCESS]... и [FAILED]...
			
Тестирование структуры данных:
	
	TC-Data-01 "Создание задачи со всеми полями":
		Шаги:
			1. в файле listOfTask.yml для задачи заполнить все поля (summary, issuetype, priotiry, description)
			
		Ожидаемый результат:
			1. задача создается в Jira, проверяем что все поля заполнены корректно
		
	TC-Data-02 "Создание задачи без опционального поля(descrition)":
		Шаги:
			1. в файле listOfTask.yml для задачи заполнить все поля кроме поля description
			2. запустить программу
			
		Ожидаемый результат:
			1. задача создается в Jira
			2. поле "Описание" остается пустым

Тестирование завершения приложения:

	TC-Exit-01 "Завершение программы при критической ошибке":
		Шаги:
			1. выполнить тесты TC-Validation-01 и TC-Validation-02
			
		Ожидаемый результат:
			1. программа завершает работу после первой же ошибки и не пытается создавать задачи
			
	TC-Exit-02 "Продолжение работы программы при ошибке в задаче":
		Шаги:
			1. в файле listOfTask.yml должно быть две задачи (одна без summary, а вторая валидная)
			2. выполнить тест TC-Validation-04
			
		Ожидаемый результат:
			1. программа пропускает первую задачу, но создает вторую
