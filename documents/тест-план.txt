Тест-план программы Jira Task Creator

1. Введение
	1.1 Цель документа
		Данный документ описывает что мы будем тестировать, как будет тестировать и какие ресурсы для этого понадобятся.
	1.2 Объект тестирования
		Объектом тестирования является Java-программа предназначенная для автоматического создания задач в системе управления проектами и отслеживания задач Jira на основе данных из yml-файла.
	1.3 Ссылки на требования
		Тестирование проводится на основании документа "Требования к проекту Jira Task Creator" (файл проект.txt версии от 22.02.2026)
		
2. Область тестирования
	2.1 Что тестируется:
		Функциональное тестирование:
			- Чтение и валидация конфигурационного файла
			- Чтение, парсинг и валидация файла с задачами listOfTask.yml
			- Корректный маппинг данных из YAML в Java POJO
			- Обработка успешных и неуспешных ответов от Jira API
			- Работа программы в различных сценариях, описанных в требованиях (пропуск задач при ошибках, заверщение работы программы при отсутствии файла)
			
		Нефункциональное тестирование:
			- Тестирование логирования: проверка создания лог-файла, формата и содержания сообщений (SUCCESS, FAILED, ERROR)
			- Тестирование задержек: проверка наличия и корректности задержки между запросами
	
	2.2 Что не тестируется:
		- Функциональность и стабильность самой системы Jira
		- Производительность Jira API
		- Безопасность инфраструктуры, на которой будет запускаться программа
		- Юзабилити (User Experience) программы, так как она не имеет графического интерфейса
		
3. Стратегия тестирования
	- модульное тестирование (покрытие основных модулей: парсер YAML, валидация полей, маппер в POJO)
		инструменты: JUnit, Mockito
	- функциональное тестирование (проверка всей сценариев использования программы, позитивных и негативных)
	- интеграционное тестирование (проверка взаимодействия с Jira API)
	- тестирование логгирования
	- тестирование обработки ошибок
	- регрессионное тестирование

4. Критерии начала и окончания тестирования
	4.1 Критерии начала тестирования:
		- завершена минимально работающая версия программы
		- пройдены модульные тесты
		
	4.2 Критерии завершения тестирования:
		- все запланированные тест-кейсы выполнены
		- в программе отсутствуют критические и высокоприоритетные ошибки
		- заведены баг-репорты на все найдены дефекты
		
5. Окружение
	- операционная система: Windows 11
	- Java: OpenJDK 17
	- система сборки: Maven 4.0
	Инструмены:
		- Postman
		- Git

6. Резульаты тестирования
	По итогам тестирования будут подготовлены следующие документы:
		- отчет о тестировании: итоги выполнения тест-плана
		- баг-репорты: детальное описание всех найденных дефектов
